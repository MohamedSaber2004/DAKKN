## Project Architecture (for reference)

### Solution Structure
```
DAKKN.sln
├── DAKKN.Domain           # Entities, Enums, Repository Interfaces
├── DAKKN.Application      # CQRS via MediatR, DTOs, Interfaces, Validators, Localization
├── DAKKN.Infrastructure   # JWT, Email, Identity setup
├── DAKKN.Persistence      # EF Core DbContext, Migrations, Seeding, Audit Interceptor
└── DAKKN.MVC              # Controllers, Views, Layouts, wwwroot
```

**Dependency direction (strict):** `Domain ← Application ← Infrastructure / Persistence ← MVC`

### Application Layer Rules
- All business logic lives in `DAKKN.Application` via MediatR Commands/Queries
- Handlers use `IUnitOfWork` — never inject `DbContext` directly
- Exceptions: only `ValidationException` (400), `BadRequestException` (400), `NotFoundException` (404), `UnAuthorizedException` (401)
- ApiResponse wrapper: `ApiResponse<TData>.Ok(data, message)`
- All user-facing strings use `IStringLocalizer<Messages>` with keys from `LocalizationKeys` static class — never hard-code raw strings

### MVC/Presentation Layer Rules
- Admin controllers: `[RoleAuthorize(UserType.Admin)]` (not `[Authorize]`)
- CSRF: `@Html.AntiForgeryToken()` on every form
- Tailwind via CDN, dark mode via `class` strategy

### Image Upload Rules (already implemented, do NOT change)
- Allowed extensions: `.jpg`, `.jpeg`, `.webp`, `.bmp`, `.png`, `.gif`, `.jfif`
- Max file size: **10 MB**
- Upload paths mapped by integer: 0=Root, 1=Products, 2=Categories, **3=Users**, 4=Invoices, 5=Reviews
- File naming: `{place}{timestamp}{ext}` (e.g., `320260618123000001.png`)
- `IImageValidator` service handles upload, delete, validation
- `CustomFileProvider` serves files at `/files/{filename}` URL
