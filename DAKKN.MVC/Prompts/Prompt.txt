Analyze the entire DAKKN repository and perform a complete authentication UI/UX theme consistency audit.

Problem:
The Landing Page supports Light Mode and Dark Mode.

When a user changes the theme on the Landing Page and then navigates to:

* Sign In
* Sign Up
* Forgot Password
* Reset Password
* Verify OTP
* Email Confirmation
* Resend Verification
* Change Password
* Account Activation
* External Login pages
* Any Authentication-related page

there are visual inconsistencies between the Landing Page theme and Authentication pages.

Goal:
Create a single, unified theme system across the entire application.

Requirements:

1. Perform a Full Theme Architecture Audit

Analyze:

* Theme management logic
* Theme persistence logic
* Theme initialization
* LocalStorage usage
* Cookie usage
* Theme synchronization
* Theme switching implementation
* Dark mode implementation
* Layout files
* Shared layouts
* Authentication layouts
* Partial views
* CSS files
* Tailwind configuration
* JavaScript theme initialization
* Razor views
* MVC layouts

Identify every place where theme handling differs.

2. Root Cause Analysis

Find all causes of inconsistency such as:

* Different layouts using different theme logic
* Missing dark classes
* Missing theme initialization
* Theme flashing during page load
* LocalStorage not read correctly
* Cookie not synchronized
* Theme loaded too late
* Authentication pages using separate CSS
* Components not supporting dark mode
* Hardcoded colors
* Missing Tailwind dark variants
* Missing CSS variables
* Different rendering pipelines

Document every issue discovered.

3. Authentication Pages Audit

Inspect all auth-related pages:

Sign In

Sign Up

Forgot Password

Reset Password

OTP Verification

Email Verification

Change Password

Account Lockout

Access Denied

External Login

Two Factor Authentication

Any Identity pages

Verify:

* Background colors
* Cards
* Inputs
* Labels
* Buttons
* Validation messages
* Icons
* Borders
* Shadows
* Hover states
* Focus states
* Error states
* Success states
* Loading states

4. Theme Synchronization

Implement a single source of truth.

Requirements:

* Landing Page theme change must instantly affect Auth pages.
* Auth pages must read the same theme preference.
* Theme must persist after:

  * Navigation
  * Refresh
  * Login
  * Logout
  * Session expiration
  * Browser restart

Verify LocalStorage, Cookies, and Database settings behavior.

5. Prevent Theme Flashing

Implement proper theme bootstrapping.

Before page render:

* Read theme immediately.
* Apply dark/light class before content paints.
* Eliminate FOUC (Flash of Unstyled Content).
* Eliminate white flash when opening auth pages from dark mode.

6. Shared Design System

Create a reusable design system.

Move all theme colors to:

* CSS variables
* Tailwind theme configuration
* Shared component styles

Remove:

* Hardcoded colors
* Duplicate styles
* Page-specific theme hacks

7. Component Consistency

Ensure identical styling behavior between:

Landing Page

Authentication Pages

Dashboard

Admin Pages

Profile Pages

Settings Pages

Checkout Pages

Any future pages

All should use the same design tokens.

8. Accessibility Review

Verify:

* Contrast ratios
* Focus visibility
* Keyboard navigation
* Error visibility
* Dark mode readability
* Placeholder visibility

9. Mobile Review

Verify theme consistency on:

* Mobile
* Tablet
* Desktop

Check responsive layouts in both light and dark modes.

10. Performance Review

Ensure:

* No duplicate theme scripts
* No duplicate CSS
* No unnecessary DOM updates
* No layout shifts
* No hydration/theme mismatch

11. Deliverables

Provide:

* Complete root cause analysis.
* All discovered inconsistencies.
* List of affected files.
* List of layout files.
* List of CSS files.
* List of JS files.
* Theme architecture diagram.
* Refactoring plan.
* Final implementation.

Important:

Do not stop after fixing the visible Sign In and Sign Up pages.

Audit the entire authentication flow and identify every possible theme inconsistency that currently exists or may occur in the future. Implement a centralized, scalable, maintainable theme system that guarantees visual consistency across the whole DAKKN application.
