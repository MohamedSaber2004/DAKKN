If you want an AI (or ChatGPT) to investigate the real root cause of images not loading across your entire ASP.NET Core MVC site, use this prompt:

Act as a Senior ASP.NET Core MVC Architect and Debugging Expert.

My project runs successfully, pages load correctly, but ALL images across the entire website are not displayed.

Your task is NOT to guess. Your task is to find the ROOT CAUSE.

Follow this investigation process:

1. Analyze the complete project structure.
2. Check Program.cs and Startup configuration.
3. Verify UseStaticFiles() middleware configuration.
4. Verify wwwroot folder existence and structure.
5. Check image physical locations.
6. Check image URLs generated in Razor Views.
7. Check Layout files (_Layout.cshtml).
8. Check image paths in:
   - img tags
   - CSS background-image URLs
   - JavaScript-generated images
9. Check environment-specific configurations.
10. Check publish/deployment settings.
11. Check IIS/Kestrel static file serving settings.
12. Check file permissions.
13. Check case-sensitivity issues.
14. Check browser Network Tab errors.
15. Check Console errors.
16. Check MIME type problems.
17. Check URL rewriting issues.
18. Check image upload and storage logic if images come from database.
19. Check database image paths.
20. Check Razor path helpers:
    - Url.Content()
    - asp-append-version
    - ~/ paths

For every issue found:

- Explain WHY it causes images not to load.
- Show evidence from the code.
- Explain how to reproduce the issue.
- Provide the exact fix.
- Provide the corrected code.

Do not stop at symptoms.

Keep investigating until the actual root cause is identified.

At the end provide:

## Root Cause
[exact reason]

## Proof
[files and code lines proving it]

## Fix
[exact code changes]

## Verification Steps
[how to confirm images are loading again]

Wait until I upload the project files before starting the investigation.

For an even stronger result, upload these files with the prompt:

Program.cs
_Layout.cshtml
Any view containing <img>
appsettings.json
Screenshot of Browser Network Tab (F12 → Network)
Screenshot of Console Tab
Screenshot of Solution Explorer showing wwwroot

This usually allows identification of the exact root cause in a few minutes rather than guessing.
