IMPORTANT ARCHITECTURE RULES

Current DAKKN architecture:

- Categories are loaded dynamically from the database.
- Category.ImageUrl stores ONLY the image file name.
- The database NEVER stores:
  - Full URLs
  - Absolute URLs
  - Physical paths
  - Relative file system paths

Example:

Database value:
Category.ImageUrl = "electronics.webp"

NOT:
"https://localhost:5001/files/electronics.webp"

NOT:
"/uploads/categories/electronics.webp"

Image files are served through the Files API endpoint:

GET {BaseUrl}/files/{imageName}

Examples:

BaseUrl:
https://localhost:5001

ImageName:
electronics.webp

Final URL:
https://localhost:5001/files/electronics.webp

Image naming convention (per Gemini.md §5.5):
- File names are prefixed with the place index (2 for Categories) and include a timestamp.
- Example: "2202506151230001.jpg" (2 = Categories place, 202506151230001 = timestamp).
- The first digit of an existing image name must match its UploadPlace.
- Allowed extensions: .jpg, .jpeg, .webp, .bmp, .png, .gif, .jfif.
- Max file size: 10 MB.
